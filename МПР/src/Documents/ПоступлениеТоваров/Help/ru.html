Поступления товаров

Документ Поступление товаров предназначен для оформления факта поступления товаров от сторонних поставщиков товаров.

В зависимости от настроек магазина могут применяться две схемы оформления поступления:

· Однофазная– менеджер оформляет документ Поступление товаров, согласно которому увеличиваются остатки товара на складе магазина;

· Двухфазная (ордерная)– менеджер вводит в систему документ Поступление товаров, в котором регистрирует информацию по ценам и количеству поступившего товара, но фактически принятое на склад количество товара регистрирует кладовщик с помощью документа Приходный ордер.

Список документов поступления можно открыть по ссылке Закупки - Закупки - Поступления товаров. Для удобства работы Пользователь может установить отбор по полям Магазин, Организация, Поставщики Ответственный в шапке списка документов. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов. Дополнительные команды доступны через кнопку Еще.

На основании документа Поступление товаров можно оформить следующие документы:

· Внутренние перемещения товаров - для перемещения поступившего товара на другой склад данного магазина.

· Возврат товаров поставщику - для оформления полного или частичного возврата поставщику товара из данной поставки.

· Выполнить приемку – с размещением

· Все количество на один склад

· Оставшееся количество на склад

· По потребностям

· По правилам распределения.

· Заявка на расходование ДС.

· Перемещение товаров - для перемещения поступившего товара в другой магазин.

· Расходный кассовый ордер - для регистрации полной или частичной оплаты поставки наличными денежными средствами.

· Регистрация безналичной оплаты - для регистрации полной или частичной безналичной оплаты поставки.

· Установка цен номенклатуры - для ценообразования вновь поступившего товара. В документ переносятся все товарные позиции документа-основания.

· Установка цен номенклатуры (по изменившимся ценам поставщика) - в документ переносятся только те товарные позиции, рассчитанные цены которых изменились более, чем на указанный в соответствующем Виде цен порог срабатывания.

На основании «Поступления товаров» могут быть напечатаны следующие документы:

· Акт расхождений,

· М-4 (Приходный ордер),

· Опись номенклатуры (список товаров без указания цен и сумм),

· Приходная накладная (по розничным и ценам поступления),

· Приходная накладная (по розничным ценам),

· Приходная накладная (по ценам поступления),

· ТОРГ-12 (Товарная накладная за поставщика с услугами),

· ТОРГ-12 (Товарная накладная за поставщика),

· ТОРГ-2 (Акт об установлении расхождения),

· ТОРГ-4 (Акт о приемке товара, поступившего без счета поставщика),

· Ценники и этикетки

По каждому документу в списке выводится дата, номер, поставщик, входящий документ и т.д. Документы, по которым просрочена оплата, отмечаются специальным значком-часами в первом поле списка. Определение статуса просроченной оплаты производится с помощью регламентного задания. Регламентное задание достаточно запускать один раз в день, перед началом рабочего дня, а также после завершения обмена РИБ, если оплата поставщикам производится также в центральном офисе. Регламентное задание настраивается на странице регламентных заданий, которую можно открыть по ссылке Администрирование - Обслуживание – Регламентные операции - Регламентные и фоновые задания. В локальной базе статусы просроченной оплаты обновляются при проведении документов оплаты.

Заполнение шапки документа

В шапке нового документа нужно заполнить следующие поля:

· Поставщик, от которого поступил товар, выбирается из списка контрагентов;

· Магазин, в который поступил товар;

· Склад, на который будет приходоваться товар, по умолчанию - склад поступления, указанный для данного магазина;

· Организация, по умолчанию подставляется организация склада.

Некоторые из этих полей заполняются автоматически в следующих случаях:

· для пользователя, создающего документ, соответствующие значения настроены по умолчанию;

· перед созданием нового документа в списке документов были установлены значения отбора;

· поступление формируется на основании документа Заказ поставщику.

Комиссионная торговля

1. В созданном документе на вкладке "Дополнительно" для реквизита "Операция" устанавливаем значение "Прием на комиссию"

2. Становится доступным реквизит "Договор", который необходимо заполнить ранее заведенным договором.

3. Осуществляется подбор товаров

4. При проведении документ создает движения по регистру "Товары организаций" заполняя измерения "Поставщик" и "Договор"

Заполнение табличной части Товары по факту

Если документ Поступление товаров формируется на основании документа Заказ поставщику, то табличная часть заполняется из документа-основания.

Добавить товары в табличную часть можно следующими способами:

· с помощью кнопки Добавить командной панели табличной части документа;

· с помощью сервиса Подобрать товары, который запускается из списка Заполнить в командной панели табличной части;

· с помощью сервиса Заполнить по ордерам, в результате выполнения которого табличная часть будет перезаписана сведениями о товаре и его количестве из документа Приходный ордер, сформированного на том же заказе-основании, что и данное поступление;

· загрузить список товаров из терминала сбора данных, в этом случае табличную часть следует предварительно очистить, т. к. иначе сведения по количеству товара из ТСД просуммируются с количеством в соответствующих товарных строках табличной части;

· по штрихкоду.

Добавление товара в табличную часть по штрихкоду

Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне.

Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1.

Для каждой товарной строки указывается:

· Характеристика номенклатуры – можно выбрать из списка характеристик данной номенклатуры или распределить с помощью сервиса Распределить по характеристикам;

· Упаковка – по умолчанию указывается основная единица измерения, но может быть выбрана любая упаковка данной номенклатуры;

· Количество – количество поступивших единиц или упаковок;

· Цена – закупочная цена товара;

· Ставка НДС – заполняется по указанной для данной номенклатуры ставке НДС.

Список Цены в командной панели табличной части содержит ссылки на сервисы, с помощью которых можно заполнить цены поступления в выделенных пользователем строках. Чтобы выделить все строки, нужно нажать комбинацию клавиш Ctrl+A. Доступны следующие сервисы:

· Заполнить по ценам прошлой поставки – цены поступления будут заполнены по ценам прошлой поставки этого товара от указанного поставщика;

· Заполнить цены по данным поставщика (при расхождении) – цены поступления будут заполнены новыми значениями цен поставщика;

· Сравнивать с ценами прошлых поставок – в табличной части будут добавлены поля для сравнения новых цен с ценами прошлой поставки.

Ввод серии товара

Серию товара необходимо вводить в строке табличной части документа Поступление товаров при выполнении нескольких условий: на предприятии ведется учет товара по сериям, для данного вида номенклатуры настроена политика учета серий, согласно этой политике нужно учитывать серии при приемке товаров от поставщика; схема поступления товаров для данного магазина однофазная.
В этом случае в строке товара в поле Количество будет отображен значок Статусы серии.
Значок красного цвета означает, что серии еще не введены.
Для ввода серии нужно из поля Количество открыть форму Ввод количества и серий товаров и заполнить её.
После корректного ввода серии и количества, значок серии меняет свой цвет на зеленый.

Регистрация расхождения между фактическим поступлением товара и указанным в сопроводительных документах

Если при пересчете поступившего товара обнаружено расхождение с сопроводительными документами, то в шапке соответствующего документа Поступление товаров нужно установить флаг Есть расхождения. При этом становится доступна дополнительная табличная часть Товары по данным поставщика, которая автоматически заполняется сведениями из табличной части Товары по факту. Пользователь может изменить в строках табличной части количество, цену и ставку НДС, следя за тем, чтобы суммы в разделе Итого по данным поставщика совпали с сопроводительными документами. В процессе редактирования пользователь может восстановить исходный список с помощью сервиса Заполнить товарами по факту меню Заполнить командной панели списка товаров по данным поставщика.

Анализ расхождений при поставке

После проведения документа по ссылке Анализ расхождений в панели навигации документа, можно просмотреть отчет о расхождении при поступлении товаров по Количеству и Сумме.

Недопоставленные товарные строки выделяются красным; товары, поступившие с излишком, - зеленым цветом. В отчете также отображаются расхождения по итоговым суммам документа.

Аналогичный отчет можно также сформировать по всем поставщикам и документам с расхождениями за определенный период. Отчет Анализ расхождений при поступлении товаров размещен в группе отчетов раздела Запасы и закупки.

Планирование оплаты поступившего товара

Если документ Поступление товаров создан на основании документа Заказ поставщику, то планирование оплат выполняется в документе-основании. Если документ Поступление товаров создан вручную, то планирование оплат можно выполнить на странице Этапы оплат.

Долг поставщика на текущий момент отображается в группе Взаиморасчеты итоговой области документа. Долг поставщика складывается из сумм предоплат, произведенных до момента создания документа Поступление товаров, и сумм недопоставленных оплаченных товаров.
Зачет предоплаты можно произвести из списка документов оплаты, который открывается по команде Зачет оплаты из командной панели документа Поступление товаров.

Пользователь может зарегистрировать планируемые платежи по поступившему товару на странице Этапы оплат. По умолчанию система предлагает оплату наличными 100% в день планируемой поставки. Пользователь может отредактировать платежи, вводя нужное количество этапов, форму оплаты, долю суммы заказа в этапе. Для каждой записи он также может изменить дату платежа или количество дней отсрочки.

Возможно, автозаполнение этапов оплат в документах Заказ поставщику и Поступление товаров по заранее созданному графику оплаты. Заполнить этапы оплаты можно выбрав любой заранее созданный график оплаты из списка или предустановленный для поставщика (закладка Этапы оплат). Заполнение этапов оплаты производится во время сохранения или проведения документа.

Взаимозачет по недопоставкам можно произвести, нажав кнопку Подобрать взаимозачет командной панели табличной части. В открывшейся форме выводится список документов данного поставщика с указанием суммы, которую можно зачесть по каждому документу. По команде Перенести в документ будут сформированы строки этапов оплат, соответствующие выбранным документам.

Дополнительные реквизиты документа Поступление товаров

На странице Дополнительно вносятся номер и дата документа поставщика, сопровождающего поставку, и устанавливаются флаги в разделе Налогообложение.
Установленный по умолчанию флаг Учитывать НДС может быть снят вручную, при этом сумма документа рассчитается без учета НДС. Признак Цена включает НДС влияет на значение цены, отображаемой в табличной части документа, и на алгоритм расчета суммы документа.

Действия системы при проведении документа

Если документ Поступление товаров был сформирован на основании документа Заказ поставщику, то при проведении документа поступления заказ-основание автоматически закрывается.

Распределение товаров по складам

Если для данного магазина установлена ордерная схема приемки товаров и товары из данного поступления нужно распределить на разные склады магазина, то для формирования приходных ордеров на товары можно воспользоваться сервисом Распределить по складам, который запускается из командной панели документа.

Анализ расхождений между заказанным и поступившим товаром

Заказанные у поставщика и поступившие от этого поставщика товары регистрируются в системе документами Заказ поставщику и Поступление товаров, соответственно. Расхождения между заказом и поставкой анализируются с помощью отчета Исполнительность поставщиков.

Учет расхождений при поступлении

В случае, если при пересчете товара выявились расхождения фактического наличия товаров с накладной, пользователь, так же, как и при приемке обычной накладной поставщика, устанавливает флаг Есть расхождения, и на закладке Товары по факту вносит изменения.

При этом формируются итоги по факту и по данным поставщика. При выполнении команды Передать в ЕГАИС данные по расхождениям будут переданы в систему ЕГАИС для подтверждения поставщиком.

Поступление КиЗ от эмитента

Поступление КиЗ от эмитента оформляется документом Поступление товаров. Документ может быть оформлен на основании с ранее оформленным документом Заказ поставщику.
Информация о полученных этикетках и номерах КиЗ регистрируется в качестве серии товаров.

В серии указывается полученные номера КиЗ. Номер КиЗ можно ввести вручную или считать с использованием сканера штрихкодов или RFID – считывателя. Считывание штрихкода или RFID–меток происходит последовательно. Для активизации работы RFID устройства необходимо использовать пиктограмму со знаком вопроса при считывании каждой этикетки. После регистрации серий необходимо провести документ. Будет зарегистрировано поступление на склад этикеток с номерами КиЗ для маркировки товаров. Если Заказы и Заявки на выпуск КиЗ для персонифицированных и не персонифицированных КиЗ оформляются раздельно, то в поступлении можно совмещать КиЗ разных типов. Для персонифицированных КиЗ информация о сериях регистрируется отдельно для каждой позиции.


После того как этикетки с указанием номера КиЗ приняты на склад, необходимо подтвердить информацию об их поступлении и передать эту информацию в программу ГИСМ. Это можно сделать из списка Заявки на выпуск КиЗ для маркировки. В списке можно выделить те заявки, для которых нужно выполнить действие Подтвердить получение и выполнить групповую команду.

Аналогичную операцию можно выполнить из формы документа Поступление товаров. Предусмотрена также возможность просмотра всех необходимых действий на рабочем месте 1С:Маркировка (раздел Закупки). После подтверждения получения КиЗов необходимо выполнить обмен с системой ГИСМ и закрыть заявки после получения подтверждения от системы ГИСМ.

Поступление маркируемой продукции

Оформление поступления товаров при отсутствии электронного УПД

Так как до июля 2020 года сопровождение поставки электронным УПД не является обязательным для всех участников оборота табачной продукции, то могут возникнуть ситуации, когда поставка прошла, а УПД в ЭДО не пришел. Документ Поступление товаров и коды маркировки не будут загружены в базу, соответственно, проверить сканированием их не удастся.

В такой ситуации можно создать документ Поступление товаров в ИБ вручную. В ТЧ документа необходимо подобрать поступившие позиции с признаком маркируемой табачной продукции (см. раздел Настройка НСИ) и провести документ. Пиктограмма  свидетельствует о том, что товар маркируемый, но она останется красного цвета, так как товар не отсканирован.

При дальнейшем оформлении продажи или возврата поставщику, данный товар необходимо отсканировать в соответствующем документе и провести его. В справочнике Штрихкоды упаковок и товаров раздела НСИ будет сохранена информация о считанных кодах маркировки.

Важно: при оформлении поступления товаров вручную, продавать и возвращать блоки можно только при включенном контроле статусов кодов маркировки. Это связано с тем, что информация о вложениях блока не была загружена из электронного УПД и ее нужно будет получить из ИС МОТП.

Оформление поступления товаров на основании электронного УПД

Поступление маркируемой продукции сопровождается электронным УПД, на основании которого в базе создается документ Поступление товаров (см. раздел ЭДО). В интерфейсе пользователя документы данного вида расположены в разделе Закупки.

Созданный документ будет заполнен по данным из УПД. Шрихкоды упаковок будут сохранены в базе в том виде, в котором они были переданы поставщиком. В ТЧ документа для маркируемых товаров отображается пиктограмма , которая свидетельствует о необходимости считывания штрихкодов упаковок. Для немаркируемых товаров данная пиктограмма не отражается.

В течение трех рабочих дней нужно подтвердить поступление товаров поставщику в ЭДО. Предварительно рекомендуется выполнить фактическую проверку поступивших кодов маркировки путем сканирования. Сканировать можно короба, блоки или отдельные пачки.

Для каждого типа упаковки предусмотрено сканирование определенного формата:

· для пачек - двумерный штрихкод формата DataMatrix;

· для блоков – двумерный штрихкод формата DataMatrix по стандартам GS1;

· для логистических (транспортных) упаковок – одномерный штрихкод формата GS1-128.

Для проверки в открытой форме документа необходимо нажать ссылку Проверить поступившую табачную продукцию. В документах поступления сканирование табачной маркировки следует производить только в открывшейся форме Проверка поступившей табачной продукции, в противном случае программа выдаст предупреждение.

При включенной опции Контролировать статусы кодов маркировки в данной форме проверки выполняется онлайн контроль статусов поступивших кодов маркировки. Например, проверка не будет пройдена, если в ИС МОТП за данной продукцией числится другой владелец. Так как до июля 2020 года оформление УПД является обязательным только между производителем и первым покупателем, то подобные «пробелы» в цепочке поставок товара могут встречаться. Поэтому, если необходимо принять товар, игнорируя предупреждение системы, можно отключить контроль, нажав кнопку Еще и выбрав вариант Отключить проверку кодов по владельцу.

Форма Проверка поступившей табачной продукции состоит из двух вкладок, каждую из которых целесообразно рассмотреть отдельно.

На вкладке Структура упаковок содержится следующая информация:

Содержимое – информация об иерархической структуре упаковок и их вложений, полученной из УПД поставщика. Для пачек, находящихся вне блоков, предназначен отдельный узел дерева Пачки без блока. Структура может быть дополнена, если будет отсканирован товар, который изначально отсутствовал в ТЧ документа.

Детализация - информация о текущей детализации отображается внизу формы в виде ссылки, при нажатии на которую открывается форма выбора уровня детализации.

При выявлении некорректного содержимого – по ссылке можно выбрать вариант поведения программы, если при проверке содержимого упаковки были выявлены некорректные вложения.

Статус проверки – информация о текущем статусе проверки. Последующие колонки содержат информацию о количестве упаковок и вложений в разных статусах:

· Не проверялась - означает, что товар должен быть среди принимаемого или отгружаемого, но его наличие еще не подтверждено.

· В наличии - наличие пачки или блока было подтверждено.

· Отсутствует - товар был признан отсутствующим.

· Отложена - продукция была отложена. Такая ситуация может возникнуть, когда пачка находится не в том блоке, в котором должна.

· Не числилась - фактическое нахождение товара не соответствует начальной информации о его местонахождении.

· Не определен – отсутствует информация о товаре.

· ∑ - общее количество проверенных элементов.

При необходимости статус можно сменить вручную, нажав кнопку Статус проверки и выбрав подходящий вариант.

На вкладке Табачная продукция отражается список товаров с указанием количества из ТЧ документа Поступление товаров и количества отсканированных позиций. Если после проверки данные совпадают, то пиктограмма  изменит цвет на зеленый. Для завершения проверки необходимо нажать кнопку Проверка завершена, после чего эти результаты будут доступны только для просмотра.

Примечание: чтобы продолжить проверку необходимо в ТЧ документа нажать кнопку Еще и выбрать вариант Возобновить проверку и подбор табличной части.

Если имеются расхождения между данными ТЧ документа и отсканированными, то после нажатия кнопки Проверка завершена ТЧ документа будет заполнена данными из формы Проверка поступившей табачной продукции.

Для товаров со статусом Отложена или Не проверялась программа задаст уточняющий вопрос об установке статуса товара. Если выбрать вариант В наличии, то товар будет отражен в ТЧ документа. Если же выбрать вариант Отсутствующие, то товар в ТЧ перенесен не будет.

При сканировании блока в строке отражается количество вложенных пачек.

Если все товары успешно прошли проверку, то в ТЧ документа пиктограмма  поменяет цвет с красного на зеленый. Проверенный документ Поступление товаров необходимо сохранить в базе. Счет-фактура оформляется стандартным способом.

Чтобы зарегистрировать информацию о смене владельца товара в ИС МОТП необходимо подтвердить УПД в ЭДО, используя УКЭП.

Примечание: на данный момент отсутствует техническая возможность учитывать расхождения при поступлении маркируемой табачной продукции, поэтому подтверждать нужно только документы без расхождений.

Перейти к работе с электронными документами можно из формы документа Поступление товаров, нажав кнопку ЭДО или ссылку внизу формы, отражающую состояние ЭДО.

В открывшейся форме нужно последовательно выбрать команды Утвердить и Подписать и отправить. Возможно, потребуется ввести пароль закрытого ключа сертификата и нажать кнопку Подписать.

Примечание: регламент ЭДО можно настроить для каждого документа учета в соответствующем профиле настроек ЭДО (Администрирование – Обмен электронными документами – Профили настроек ЭДО).
